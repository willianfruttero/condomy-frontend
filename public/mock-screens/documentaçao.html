1. Visão Geral

Condomy é a plataforma SaaS multi-tenant para gestão completa de condomínios:

  • Administração financeira e emissão de boletos;
  • Comunicação interna entre moradores;
  • Reservas de áreas comuns;
  • Dashboards e relatórios de performance.

2. Stack Tecnológico

  • Frontend: Next.js (SSR/SSG), React 18, TypeScript, Styled-Components, Tailwind CSS, Vite, React Query, React Hook Form, Zod, Axios, Lucide React, Chart.js, TanStack Table
  • Backend: Java 17, Spring Boot 3, Spring Cloud Gateway/Config, Spring Data JPA + PostgreSQL, RabbitMQ, Swagger/OpenAPI
  • Infraestrutura: Docker, Docker Compose, Kubernetes (EKS), AWS (EC2, RDS, S3), Terraform
  • CI/CD & Observabilidade: GitHub Actions, SonarCloud, ELK Stack, Prometheus + Grafana

5. Backend

  • Monorepo de microserviços Spring Boot (api-gateway, auth, condominium, communication, finance).
  • common-lib para DTOs e utilitários compartilhados.
  • JWT + RBAC (Spring Security) e validações com Zod no frontend.
  • Persistência PostgreSQL + Flyway e mensageria RabbitMQ.
  • Configuração central (Spring Cloud Config) e discovery via Eureka.

6. Frontend

  • Next.js (app & page router) com SSR/SSG e prefetching.
  • React 18 + TypeScript + Styled-Components + Tailwind CSS.
  • State global via Context API (AuthContext, SidebarContext, NotificationContext).
  • Fetch & cache com React Query; formulários com React Hook Form + Zod.
  • Testes unitários e de integração (Jest, React Testing Library).

7. API Contracts

Swagger UI em cada serviço: /swagger-ui.html; JSON em /v3/api-docs. Gateway consolida em /actuator.

  • POST /auth/login — retorna JWT.
  • GET /condominio/{id}/unidades — lista unidades.
  • POST /comunicados — cria comunicado.
  • POST /finance/payments — registra pagamento.

8. Integrações de Boletos

Gerencianet & CobreBem

  • SDKs para emissão, consulta, cancelamento e webhooks em /finance/webhook.

Itaú Automação

  • Certificado PJ e credenciais no Developer Portal Itaú.
  • Remessa CNAB240 e retorno “nosso número”.

Banco Inter Automação

  • App no Internet Banking (Cobrança, Pix), upload de chaves/certificados.
  • Até 30 boletos grátis; tarifa por liquidação a partir daí.

9. CI/CD & Deploy

  • GitHub Actions: lint, build, testes, coverage ≥ 80%.
  • Backend: Docker → registry → Terraform → EKS.
  • Frontend: Next.js → Vercel ou S3 + CloudFront.
  • Quality Gates: SonarCloud, scan de vulnerabilidades.

10. Boas Práticas

  • Commits semânticos (Conventional Commits).
  • Pull request com code review obrigatório e branches protegidos.
  • Lint & Prettier no pre-commit (Husky).
  • Testes unitários e de integração em todos os serviços.
  • Swagger sempre atualizado e disponível no pipeline.
  • Mock-screens para validar UI antes da implementação.

11. Glossário

  • JWT: JSON Web Token para autenticação.
  • RBAC: Controle de Acesso Baseado em Funções.
  • SSR: Server-Side Rendering.
  • SSG: Static-Site Generation.
  • Webhook: Callback HTTP para eventos em tempo real.
  • CNAB240: Layout de remessa/retorno bancário.
  • Micro-tenant: Isolamento de dados por cliente.

12. Administração Global & Gestão de Acessos

Papéis

  • Administrador Global: dono da plataforma; cria condomínios, define planos e libera acessos.
  • Síndico (Admin de Condomínio): gerencia residentes, finanças e comunicação do próprio condomínio.
  • Morador: consome comunicados, boletos, reservas e histórico financeiro.

Fluxo de Onboarding

  1. Global Admin cria condomínio e escolhe plano (Básico/Pro/Enterprise).
  2. Convite enviado ao Síndico por e-mail com link seguro.
  3. Síndico completa cadastro e paga assinatura (boleto/cartão).
  4. Acesso total liberado após confirmação de pagamento.

Modelos de Assinatura

  • Básico: até 50 unidades — R$ 100/mês.
  • Profissional: até 200 unidades — R$ 250/mês.
  • Enterprise: ilimitado — valor sob consulta.

Dashboards

  • Global Admin: MRR, churn, número de condomínios ativos e receita por plano.
  • Condomínio: inadimplência, fluxo de caixa, uso de espaços e engagement de comunicados.
